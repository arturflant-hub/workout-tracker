# Дневник тренировок — Логика расчётов и формулы
> Документ для разработки мобильного приложения на базе Google Таблицы «Дневник тренировок»

---

## 1. Архитектура данных (листы)

| Лист | Назначение |
|---|---|
| **PROGA A** | Журнал тренировок программы A (основной лог) |
| **PROGA B** | Журнал тренировок программы B (основной лог) |
| **DATABASE** | Справочник упражнений |
| **DATABASE1** | История всех записей (агрегированная БД для поиска) |
| **ТРЕНИРОВКА СЕГОДНЯ** | Интерфейс текущей тренировки (поиск прошлых данных + ввод) |
| **DASHBOARD** | Сводные метрики и графики прогресса |
| **BODY TRACKER** | Антропометрические замеры и % жира |
| **СПРАВОЧНИК** | Вспомогательные таблицы |

## 2. Структура записи тренировки (PROGA A / PROGA B)

Каждая строка = один подход одного упражнения.

| Колонка | Поле | Тип | Описание |
|---|---|---|---|
| A | Дата | Дата | Дата тренировки |
| B | Упражнение | Текст | Название упражнения (выпадающий список) |
| C | Вес (кг) | Число | Рабочий вес в кг |
| D | Подходы | Число | Количество подходов |
| E | Повторы | Число | Количество повторений в подходе |
| F | RIR | Число | Reps In Reserve — повторы «в запасе» до отказа |
| G | Объём | Вычислено | Тренировочный объём (авто) |
| H | ПОРА + ВЕС? | Вычислено | Флаг готовности повышать вес (авто) |
| I | Комментарий | Текст | Свободный комментарий пользователя |
| J | e1RM | Вычислено | Расчётный 1-повторный максимум (авто) |
| K | Следующий вес | Вычислено | Рекомендуемый вес на следующую тренировку (авто) |
| L | Рекомендация | Вычислено | Текстовая рекомендация по прогрессии (авто) |
| M | Статус прогресса | Вычислено | Детектор плато (авто) |
| N | Целевой диапазон | Текст | Целевой диапазон повторений (задаётся вручную, напр. «8–12») |

## 3. Формулы расчёта показателей тренировки

### 3.1 Тренировочный объём (Колонка G)

**Формула:** `Объём = Вес × Подходы × Повторы`

```
G = IF(AND(C<>"", D<>"", E<>""), C * D * E, "")
```

**Логика:** Рассчитывается только если заполнены все три поля (вес, подходы, повторы). Иначе — пустая ячейка.

**Пример:** Вес=50 кг, Подходы=3, Повторы=10 → Объём = 1500 кг

---

### 3.2 Флаг повышения веса (Колонка H — «ПОРА + ВЕС?»)

**Условие:** RIR ≤ 2 И Повторы ≥ 10

```
H = IF(AND(F<>"", F<=2, E>=10), "✅ УВЕЛИЧИТЬ ВЕС", "")
```

**Логика:**
- Если у атлета остаётся 2 или меньше повторов в запасе (RIR ≤ 2) **и** он выполнил 10+ повторений — сигнал, что вес слишком лёгкий.
- Флаг не показывается, если RIR не заполнен.

---

### 3.3 Расчётный 1ПМ / e1RM (Колонка J)

**Формула Эпли:**

```
e1RM = Вес × (1 + Повторы / 30)
```

```
J = IF(AND(C<>"", E<>""), C * (1 + E / 30), "")
```

**Пример:** Вес=60 кг, Повторы=8 → e1RM = 60 × (1 + 8/30) = 60 × 1.267 = 76 кг

**Назначение:** Оценка максимальной силы без выполнения одноповторного максимума. Используется для отслеживания динамики силовых показателей.

---

### 3.4 Рекомендуемый следующий вес (Колонка K)

**Логика прогрессии на основе RIR и Повторов:**

```
K = IF(C="", "",
      IF(F<=1,                                  → вес слишком тяжёлый
         C - 5,                                 → уменьшить на 5 кг
         IF(AND(F=2, E>=верхняя_граница_диапазона),
            C + 2.5,                            → увеличить на 2.5 кг
            IF(AND(F=2, E<верхняя_граница_диапазона),
               C,                               → оставить вес
               IF(F>=3,
                  C + 2.5,                      → увеличить на 2.5 кг
                  C)))))                        → оставить вес
```

**Полная формула:**
```
K = IF(C="","",IF(F<=1, C-5,
      IF(AND(F=2, E>=VALUE(RIGHT(N, LEN(N)-FIND("–",N)))), C+2.5,
        IF(AND(F=2, E<VALUE(RIGHT(N, LEN(N)-FIND("–",N)))), C,
          IF(F>=3, C+2.5, C)))))
```

**Таблица решений:**

| RIR | Повторы vs. диапазон | Действие | Изменение веса |
|---|---|---|---|
| ≤ 1 | любые | Слишком тяжело | −5 кг |
| 2 | ≥ верхней границы диапазона | Прогресс достигнут | +2.5 кг |
| 2 | < верхней границы диапазона | Продолжать работу | 0 кг |
| ≥ 3 | любые | Слишком легко | +2.5 кг |

**Примечание:** Верхняя граница диапазона извлекается из колонки N (например, из «8–12» берётся число 12) через текстовую функцию `VALUE(RIGHT(N, LEN(N)-FIND("–",N)))`.

---

### 3.5 Рекомендация по прогрессии (Колонка L)

**4-уровневая классификация на основе RIR и Повторов:**

```
L = IF(F="", "",
      IF(F<=1,                     "⚠ УМЕНЬШИТЬ ВЕС",
        IF(AND(F=2, E>=12),        "⬆ УВЕЛИЧИТЬ ВЕС",
          IF(AND(F=2, E<12),       "👍 ОСТАВИТЬ ВЕС",
            IF(F>=3,               "⬆ МОЖНО ДОБАВИТЬ", "")))))
```

**Классификация:**

| RIR | Повторы | Рекомендация | Смысл |
|---|---|---|---|
| ≤ 1 | любые | ⚠ УМЕНЬШИТЬ ВЕС | Вес чрезмерен, риск травмы |
| 2 | ≥ 12 | ⬆ УВЕЛИЧИТЬ ВЕС | Достигнута верхняя граница диапазона с хорошим запасом |
| 2 | < 12 | 👍 ОСТАВИТЬ ВЕС | Работа идёт в целевой зоне |
| ≥ 3 | любые | ⬆ МОЖНО ДОБАВИТЬ | Слишком большой запас, вес занижен |

---

### 3.6 Детектор плато (Колонка M — «Статус прогресса»)

**Условие:** Вес не менялся 4 и более тренировок подряд.

```
M = IF(C="", "",
      IF(COUNTIF(C[текущая строка - N]:C[текущая строка], MAX(C[диапазон])) >= 4,
         "⚠ ПЛАТО", ""))
```

**Полная формула (скользящее окно):**
```
M = IF(C3="","", IF(COUNTIF(C2:C3, MAX(C2:C3))>=4, "⚠ ПЛАТО", ""))
```

**Логика:** Если максимальный вес в скользящем окне встречается 4+ раз подряд — плато. Сигнал пользователю о необходимости изменить программу.

## 4. Логика экрана «Тренировка сегодня»

Экран позволяет выбрать тип тренировки и упражнение, автоматически подтягивает исторические данные и рекомендации.

### 4.1 Входные параметры (ручной ввод пользователя)

| Ячейка | Поле | Описание |
|---|---|---|
| B3 | Тип тренировки | Выбор: A или B |
| B5 | Упражнение | Выбор из справочника |
| B16 | Вес | Вес для текущей тренировки |
| B17 | Подходы | Подходы для текущей тренировки |
| B18 | Повторы | Повторы для текущей тренировки |
| B19 | RIR | RIR для текущей тренировки |

### 4.2 Автоподтягивание исторических данных из DATABASE1

Все поля ищут **последнюю запись** по упражнению и типу тренировки:

```
Прошлый вес (B7)     = IFERROR(INDEX(FILTER(DATABASE1!D:D, DATABASE1!A:A=B3, DATABASE1!C:C=B5), 1), "-")
Прошлые подходы (B8) = IFERROR(INDEX(FILTER(DATABASE1!E:E, DATABASE1!A:A=B3, DATABASE1!C:C=B5), 1), "-")
Прошлые повторы (B9) = IFERROR(INDEX(FILTER(DATABASE1!F:F, DATABASE1!A:A=B3, DATABASE1!C:C=B5), 1), "-")
Прошлый RIR (B10)    = IFERROR(INDEX(FILTER(DATABASE1!G:G, DATABASE1!A:A=B3, DATABASE1!C:C=B5), 1), "-")
```

**Логика фильтра:** ищет строки в DATABASE1, где:
- колонка A (тип тренировки) = выбранный тип (B3)
- колонка C (упражнение) = выбранное упражнение (B5)

Берёт первое найденное значение (последняя тренировка, если данные отсортированы по убыванию даты).

### 4.3 Рекомендуемый вес (B12)

Подтягивается из колонки L (Следующий вес) последней записи:

```
B12 = IFERROR(INDEX(FILTER(DATABASE1!L:L, DATABASE1!A:A=B3, DATABASE1!C:C=B5), 1), "-")
```

### 4.4 Целевые повторы (B13)

Динамически рассчитывается на основе прошлого RIR:

```
B13 = IFERROR(
        IF(B10="-",  "10-12",
          IF(B10=1,  "8-9",
            IF(B10=2,"9-11",
                     "10-12"))),
        "10-12")
```

**Таблица целевых повторов:**

| Прошлый RIR | Целевые повторы |
|---|---|
| Нет данных | 10–12 |
| 1 (очень тяжело) | 8–9 (уменьшаем повторы — вес вырастет) |
| 2 (нормально) | 9–11 (рабочая зона) |
| 3+ (легко) | 10–12 (добавить повторы или вес) |

## 5. Сводные метрики (DASHBOARD)

### 5.1 Тренировочные показатели

| Метрика | Формула |
|---|---|
| Общий тренировочный тоннаж | `SUM(PROGA_A!G:G) + SUM(PROGA_B!G:G)` |
| Количество тренировок (записей) | `COUNTA(PROGA_A!A:A) + COUNTA(PROGA_B!A:A)` |
| Средний RIR | `IFERROR(AVERAGE(PROGA_A!F:F, PROGA_B!F:F), 0)` |
| Средний объём на тренировку | `IFERROR(Тоннаж / Кол-во записей, 0)` |
| Средний рабочий вес | `IFERROR(Тоннаж / Кол-во записей, 0)` |

### 5.2 Метрики тела (из BODY TRACKER)

```
Текущий вес        = IFERROR(INDEX(BODY_TRACKER!K:K, MAX(FILTER(ROW(K:K), K:K<>""))), 0)
Изменение веса     = IFERROR(INDEX(BODY_TRACKER!K:K, MAX(FILTER(ROW(K:K), K:K<>""))), 0)  → последняя строка
Изменение талии    = IFERROR(INDEX(BODY_TRACKER!L:L, MAX(FILTER(ROW(L:L), L:L<>""))), 0)
Текущий % жира     = IFERROR(INDEX(BODY_TRACKER!J:J, MAX(FILTER(ROW(J:J), J:J<>""))), 0)
```

**Паттерн получения последнего значения:**
```
INDEX(колонка, MAX(FILTER(ROW(колонка), колонка<>"")))
```
Находит номер последней непустой строки в колонке и возвращает её значение.

## 6. Body Tracker — Антропометрия

### 6.1 Структура записи

| Колонка | Поле | Тип |
|---|---|---|
| A | Дата | Дата |
| B | Вес (кг) | Число |
| C | Талия (см) | Число |
| D | Грудь (см) | Число |
| E | Бёдра (см) | Число |
| F | Бедро (см) | Число |
| G | Рука (см) | Число |
| H | Рост (см) | Число |
| I | Талия/Рост | Вычислено |
| J | % жира (Navy) | Вычислено |
| K | Изм. веса от старта | Вычислено |
| L | Изм. талии от старта | Вычислено |
| M | Фото | Ссылка |
| O | Шея (см) | Число (ввод) |

### 6.2 Индекс Талия/Рост (Колонка I)

```
I = IF(AND(C<>"", H<>""), C / H, "")
```

**Норма:** < 0.5 — хороший показатель здоровья.

### 6.3 Процент жира — метод ВМС США (Navy Method) (Колонка J)

**Применяется для мужчин.**

```
% жира = 86.01 × LOG10(Талия − Шея) − 70.041 × LOG10(Рост) + 36.76
```

```
J = IF(AND(C<>"", O<>"", H<>""),
       86.01 * LOG10(C - O) - 70.041 * LOG10(H) + 36.76, "")
```

**Входные данные:**
- C = Талия (см)
- O = Шея (см)
- H = Рост (см)

**Пример:** Талия=87, Шея=35, Рост=178 → % жира ≈ 26.7%

### 6.4 Изменение веса от старта (Колонка K)

```
K[строка N] = IF(B[N]="", "", B[N] - B[первая строка])
```

```
K3 = IF(B3="", "", B3 - $B$2)
```

Базовая строка ($B$2) — первая запись (абсолютная ссылка). Все последующие измерения сравниваются с ней.

### 6.5 Изменение талии от старта (Колонка L)

```
L3 = IF(C3="", "", C3 - $C$2)
```

Аналогично изменению веса, но для обхвата талии.

## 7. Концепция прогрессии нагрузки

### Принцип RIR (Reps In Reserve)

RIR — количество повторений, которые атлет мог бы ещё выполнить до отказа.

| RIR | Интерпретация | Зона |
|---|---|---|
| 0 | Отказ | Чрезмерная нагрузка |
| 1 | Почти отказ | Тяжёлая работа |
| 2 | 2 повтора в запасе | Оптимальная рабочая зона |
| 3 | 3 повтора в запасе | Умеренная нагрузка |
| 4+ | Много в запасе | Лёгкая нагрузка / разминка |

### Алгоритм прогрессии (сводная схема)

```
ЕСЛИ RIR <= 1:
    → уменьшить вес на 5 кг (перегрузка)

ИНАЧЕ ЕСЛИ RIR == 2:
    ЕСЛИ повторы >= верхней_границы_диапазона:
        → увеличить вес на 2.5 кг (прогресс)
    ИНАЧЕ:
        → оставить вес (продолжить)

ИНАЧЕ ЕСЛИ RIR >= 3:
    → увеличить вес на 2.5 кг (слишком легко)
```

### Целевой диапазон повторений (Колонка N)

Задаётся вручную в формате `«мин–макс»`, например `«8–12»`. Используется формулой выбора следующего веса для определения верхней границы прогрессии.

## 8. Связи между листами (Data Flow)

```
PROGA A / PROGA B
    │
    ├──► DATABASE1 (агрегированная история всех записей)
    │         │
    │         └──► ТРЕНИРОВКА СЕГОДНЯ
    │                   (подтягивает последние данные по упражнению)
    │
    └──► DASHBOARD
              (суммирует тоннаж, считает средние, строит графики)

BODY TRACKER
    └──► DASHBOARD
              (последние значения веса, талии, % жира)
```

## 9. Бизнес-правила для мобильного приложения

### 9.1 Обязательные поля для расчёта

Для корректного расчёта всех показателей **обязательны**:
- Вес (C)
- Подходы (D)
- Повторы (E)
- RIR (F)

Без RIR не рассчитываются: флаг повышения веса, рекомендация, следующий вес.

### 9.2 Шаг изменения веса

- Увеличение: **+2.5 кг**
- Уменьшение: **−5 кг**

Шаги фиксированы. В мобильном приложении могут быть сделаны настраиваемыми.

### 9.3 Порог плато

Плато = одинаковый вес **4 или более тренировок подряд** для одного упражнения.

### 9.4 Формат целевого диапазона повторений

В таблице используется строка вида `«8–12»` (с тире-дефисом `–`). При разработке приложения рекомендуется хранить как два отдельных поля: `rep_min` и `rep_max`.

### 9.5 % жира — формула только для мужчин

Метод ВМС США (Navy Method) применяется в данном шаблоне для мужчин. Для женщин формула отличается (учитывает обхват бёдер) и должна быть добавлена отдельно при необходимости.

### 9.6 e1RM — только оценочная метрика

Формула Эпли даёт приблизительный 1ПМ. Точность выше при количестве повторений 1–10. При 15+ повторениях погрешность возрастает.

## 10. Краткий справочник формул

| Показатель | Формула | Поля |
|---|---|---|
| Объём | `C × D × E` | Вес, Подходы, Повторы |
| e1RM (Эпли) | `C × (1 + E / 30)` | Вес, Повторы |
| % жира Navy (муж.) | `86.01 × log10(Талия−Шея) − 70.041 × log10(Рост) + 36.76` | Талия, Шея, Рост |
| Талия/Рост | `Талия / Рост` | Талия, Рост |
| Изм. от старта | `Текущее − Первое измерение` | Любой замер |
| Средний объём | `Суммарный тоннаж / Кол-во записей` | G, A |
| Следующий вес | `Таблица решений по RIR + Повторы vs. диапазон` | C, E, F, N |
